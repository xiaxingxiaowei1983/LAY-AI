## 解决方案：添加API Key设置界面

### 问题分析
- 浏览器安全限制导致无法直接读取本地.env文件
- 当前项目使用简单的HTML结构和本地HTTP服务器
- 需要一个安全、方便的方式来设置和存储API Key

### 解决方案
为HATWIN.html添加一个简单的API Key设置界面，使用localStorage存储API Key，这样用户就不需要使用浏览器控制台了。

### 实施步骤

1. **添加API Key设置模态框**
   - 在HTML中添加一个模态框，用于输入和设置API Key
   - 包含输入框、保存按钮和取消按钮
   - 只有在API Key未设置时显示

2. **使用localStorage存储API Key**
   - 修改代码，从localStorage读取API Key
   - 保存API Key到localStorage，实现持久化存储
   - 每次页面加载时自动读取

3. **更新API调用逻辑**
   - 修改ALI_API_CONFIG，优先从localStorage读取API Key
   - 保持从window.ALI_API_KEY读取的能力，作为备选方案

4. **添加设置按钮**
   - 在界面中添加一个设置按钮，用于重新打开API Key设置模态框
   - 方便用户随时更新API Key

5. **提供清晰的使用说明**
   - 在模态框中添加使用说明
   - 解释API Key的重要性和安全注意事项

### 优点
- **无需浏览器控制台**：用户可以通过界面直接设置API Key
- **持久化存储**：API Key保存在localStorage中，刷新页面后仍然有效
- **相对安全**：API Key只存储在用户本地浏览器中，不会上传到服务器
- **操作简单**：用户界面友好，操作直观
- **兼容性好**：保持与现有代码的兼容性

### 实施后效果
- 用户打开页面时，如果API Key未设置，会自动显示设置界面
- 用户输入API Key后，系统会保存并自动使用
- 用户可以通过设置按钮随时更新API Key
- 页面刷新后API Key仍然有效，无需重新设置
